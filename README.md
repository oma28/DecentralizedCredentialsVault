# Vault - Децентрализованное хранилище резервных копий

DApp для безопасного хранения зашифрованных резервных копий конфиденциальных данных с использованием блокчейна Ethereum и IPFS.

## Описание

Проект представляет собой децентрализованное веб-приложение (dApp) для хранения резервных копий секретных данных. Архитектура включает:

- **Frontend**: React 19 + Vite
- **Web3**: ethers.js v6 для взаимодействия с блокчейном
- **Блокчейн**: Ethereum (тестовая сеть Sepolia)
- **Хранилище**: IPFS через Pinata
- **Шифрование**: AES-GCM (клиентское)
- **Маршрутизация**: React Router DOM
- **Смарт-контракт**: Solidity 0.8.20 с RBAC (Role-Based Access Control)

### Ключевые особенности

- **Клиентское шифрование** - данные шифруются на стороне клиента перед отправкой
- **Децентрализация** - хранение зашифрованных данных в IPFS, метаданные в блокчейне
- **Контроль доступа** - мастер-адрес управляет списком авторизованных пользователей
- **Неизменяемость** - все операции фиксируются в блокчейне Ethereum
- **Опциональное редактирование** - возможность обновления описания записи

## Архитектура

### Смарт-контракт `Vault`

Функции контракта:
- `createBackup()` - создание новой резервной копии
- `updateBackup()` - обновление существующей копии (если разрешено)
- `deprecateBackup()` - пометка копии как устаревшей
- `authorize()` / `deauthorize()` - управление доступом (только master)
- `getBackup()` / `getAllKeys()` - получение данных о бэкапах

### Роли пользователей

| Роль | Права |
|------|-------|
| **Master** | Авторизация/деавторизация пользователей, просмотр всех записей |
| **Authorized** | Создание, чтение, обновление, удаление своих записей |
| **Unauthorized** | Только подключение кошелька, нет доступа к операциям |

### Криптография

1. **Генерация ключа**: Подпись сообщения `personal_sign` → хеширование через keccak256
2. **Шифрование**: AES-GCM с случайным IV (12 байт)
3. **Хранение**: Зашифрованные данные загружаются в IPFS, CID сохраняется в блокчейне

## Установка и запуск

### Требования

- Node.js ≥ 18
- npm
- Браузер с поддержкой Web3 (MetaMask или аналог)
- Аккаунт Pinata (для IPFS)

### Шаг 1: Клонирование репозитория

```bash
git clone <repository-url>
cd <project-directory>
```

### Шаг 2: Установка зависимостей

```bash
npm install
```

### Шаг 3: Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```env
VITE_PINATA_PROJECT_ID=id_jwt_токена_pinata
VITE_PINATA_PROJECT_JWT=jwt_токен_pinata
VITE_VAULT_CONTRACT=адрес_контракта_vault
```

**Получение JWT токена Pinata:**
1. Зарегистрируйтесь на [pinata.cloud](https://www.pinata.cloud/)
2. Создайте API Key с правами на pinning
3. Скопируйте JWT токен

**Развёртывание контракта:**
1. Используйте Remix IDE 
2. Скомпилируйте `contracts/Vault.sol`
3. Задеплойте в сеть Sepolia
4. Скопируйте адрес контракта

### Шаг 4: Запуск приложения

#### Режим разработки

```bash
npm run build
npm run dev
```

Приложение будет доступно по адресу `http://localhost:5173`

### Шаг 5: Docker (опционально)

```bash
# Сборка и запуск через docker-compose
docker-compose up --build

# Приложение доступно на http://localhost:3000
```

## Важные замечания

1. **Безопасность**: Ключи шифрования генерируются на основе подписи пользователя и никогда не передаются на сервер
2. **Сеть**: По умолчанию используется Sepolia testnet - убедитесь, что ваш кошелек подключён к правильной сети
3. **Gas fees**: Операции записи в блокчейн требуют ETH для оплаты комиссий
4. **IPFS**: Данные хранятся в IPFS через Pinata - убедитесь, что у вас есть активный аккаунт
